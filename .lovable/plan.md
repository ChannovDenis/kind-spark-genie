
# Создание учётных данных для разработчиков

## Проблема
Текущий пароль `password123` не соответствует требованиям валидации (отсутствует заглавная буква), поэтому разработчики не могут войти через форму.

## Решение
Обновить пароли существующих тестовых пользователей на соответствующие новым требованиям.

### Новые учётные данные для разработчиков

| Email | Пароль | Роль |
|-------|--------|------|
| `newuser@test.com` | `Password123` | super_admin |
| `test@example.com` | `Password123` | partner_admin |

**Новый пароль `Password123` соответствует всем требованиям:**
- 11 символов (минимум 8) ✅
- Заглавная буква `P` ✅
- Строчные буквы `assword` ✅
- Цифры `123` ✅

## Технические шаги

1. **Создать Edge Function** для обновления паролей пользователей через Supabase Admin API
2. **Вызвать функцию** для обновления паролей обоих тестовых пользователей
3. **Проверить вход** с новыми учётными данными

## Альтернативный вариант

Если не нужно менять пароли, можно **смягчить требования валидации** на клиенте (убрать проверку заглавных букв), но это снизит безопасность.

---

**Рекомендуемый вариант**: Обновить пароли до `Password123` для соответствия требованиям безопасности.
